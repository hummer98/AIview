# Implementation Plan

## Tasks

- [x] 1. FolderScannerにサブディレクトリ探索機能を追加
- [x] 1.1 (P) サブディレクトリ探索付きスキャンメソッドを実装
  - フィルター適用時に親フォルダと1階層下のサブディレクトリを探索する機能を追加
  - 探索深度を1階層のみに制限（2階層以上は探索しない）
  - 対応画像拡張子（jpg, jpeg, png, heic, webp, gif）のみをフィルタリング
  - 隠しファイル・隠しフォルダをスキップ
  - `onSubdirectories`コールバックを実装し、発見したサブディレクトリURLを通知
  - _Requirements: 1.1, 1.2, 1.3, 4.2_

- [x] 1.2 (P) 最初の画像発見時の即時コールバックを実装
  - サブディレクトリスキャン時も最初の画像発見時点で即座にコールバック
  - スキャン完了を待たずに画像表示を開始できるようにする
  - _Requirements: 4.1_

- [x] 2. FavoritesStoreに複数フォルダのお気に入り統合機能を追加
- [x] 2.1 (P) 複数フォルダのお気に入り並列読み込みを実装
  - 親フォルダと複数サブディレクトリの`.aiview/favorites.json`をTaskGroupで並列読み込み
  - 統合されたお気に入りデータをフォルダ別に管理（フォルダURL→ファイル名→レベル）
  - 個別フォルダの読み込み失敗時は空の辞書として扱い、他のフォルダは正常に処理
  - 統合モードと単一フォルダモードの両方をサポート
  - _Requirements: 2.1, 2.2, 4.3_

- [x] 2.2 (P) フォルダ別のお気に入り保存機能を実装
  - 画像URLからフォルダURLを導出し、該当フォルダの`.aiview/favorites.json`に保存
  - 既存の`favorites.json`フォーマット（ファイル名→レベルのマッピング）を維持
  - 統合モード中でも正しいフォルダに保存されることを保証
  - _Requirements: 2.3, 2.4_

- [x] 3. ImageBrowserViewModelにサブディレクトリ対応フィルタリングを追加
- [x] 3.1 サブディレクトリモード状態管理を実装
  - サブディレクトリスキャンモードの有効/無効を管理するプロパティを追加
  - 親フォルダ直下の画像URL（フィルター解除時の復元用）を保持
  - 発見したサブディレクトリURLを管理
  - 統合されたお気に入りデータの参照を追加
  - _Requirements: 1.1_

- [x] 3.2 フィルター適用時のサブディレクトリスキャントリガーを実装
  - フィルター適用時（SHIFT+1〜5）にサブディレクトリスキャンを開始
  - スキャン完了後に複数フォルダのお気に入りを統合読み込み
  - 統合されたお気に入り情報に基づいてフィルタリングを実行
  - 依存: 1.1, 1.2, 2.1 の完了が必要
  - _Requirements: 1.1, 3.1_

- [x] 3.3 フィルター解除時の状態復元を実装
  - フィルター解除（SHIFT+0）時にサブディレクトリ画像をリストから除外
  - 親フォルダ直下の画像のみの表示に復帰
  - 親フォルダのお気に入り情報のみを保持する状態に復帰
  - 現在表示中画像がサブディレクトリの場合は親フォルダの先頭に移動
  - 依存: 3.1, 3.2 の完了が必要
  - _Requirements: 1.4, 5.1, 5.2_

- [x] 3.4 サブディレクトリ対応ナビゲーションを実装
  - フィルタ条件に合致する画像のみをナビゲーション対象とする
  - サブディレクトリを含む全画像を対象としたプリフェッチを行う
  - 依存: 3.2 の完了が必要
  - _Requirements: 3.2, 3.3_

- [x] 3.5 サブディレクトリ画像のお気に入り設定を実装
  - サブディレクトリ画像にお気に入りを設定した際、該当フォルダに保存
  - お気に入り変更時にフィルタ結果を即座に再計算
  - 依存: 2.2, 3.2 の完了が必要
  - _Requirements: 2.3, 3.4_

- [x] 3.6 別フォルダオープン時のフィルター状態リセットを実装
  - 別のフォルダを開いた際にサブディレクトリモードとフィルター状態をリセット
  - 依存: 3.1 の完了が必要
  - _Requirements: 5.3_

- [x] 4. テストとバリデーション
- [x] 4.1 FolderScannerのサブディレクトリスキャンテストを作成
  - サブディレクトリが正しく探索されることを検証
  - 2階層以上の深い階層がスキップされることを検証
  - 隠しフォルダがスキップされることを検証
  - 最初の画像発見時にコールバックが呼ばれることを検証
  - 依存: 1.1, 1.2 の完了が必要
  - _Requirements: 1.1, 1.2, 1.3, 4.1, 4.2_

- [x] 4.2 FavoritesStoreの統合読み込みテストを作成
  - 複数フォルダのデータが正しく統合されることを検証
  - 並列読み込みが正しく動作することを検証
  - 正しいフォルダに保存されることを検証
  - 統合モード中のお気に入りレベル取得が正しいことを検証
  - 依存: 2.1, 2.2 の完了が必要
  - _Requirements: 2.1, 2.2, 2.3, 4.3_

- [x] 4.3 統合テストを実施
  - フォルダオープン→フィルター適用→サブディレクトリスキャン→お気に入り統合のフロー検証
  - サブディレクトリ画像にお気に入り設定→該当フォルダに保存→再読み込みで復元のフロー検証
  - フィルター適用→ナビゲーション→フィルター解除→親フォルダ直下のみ表示のフロー検証
  - 依存: すべてのタスク完了後
  - _Requirements: 1.1, 1.4, 2.1, 2.2, 2.3, 3.1, 3.2, 5.1, 5.2_
