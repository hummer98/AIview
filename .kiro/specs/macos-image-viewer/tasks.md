# Implementation Plan

## Tasks

### Phase 1: Foundation Layer

- [x] 1. プロジェクト基盤とデータ層の構築
- [x] 1.1 (P) Xcodeプロジェクトの初期設定
  - macOS 14+向けSwiftUIアプリケーションプロジェクトを作成
  - Swift Concurrencyを有効化（async/await、actor対応）
  - 必要なエンタイトルメント（ファイルアクセス、ゴミ箱操作）を設定
  - os_logによるロギング基盤を構築
  - _Requirements: 11.1, 11.2_

- [x] 1.2 (P) ファイルシステムアクセス機能の実装
  - ファイル属性（サイズ、作成日時、更新日時）取得機能
  - ゴミ箱へのファイル移動機能
  - フォルダアクセス権限の確認機能
  - アクセス権限エラー時の適切なエラーハンドリング
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 11.2_

- [x] 1.3 (P) 最近開いたフォルダ履歴の永続化
  - Security-Scoped Bookmark Dataの生成（`url.bookmarkData(options: .withSecurityScope)`）
  - UserDefaultsにBookmark Data（URL文字列ではなく）を保存・読み込み
  - 最大10件の履歴管理（古いものから削除）
  - アプリ再起動後のBookmarkからのURL復元とアクセス権限回復
  - `startAccessingSecurityScopedResource()` / `stopAccessingSecurityScopedResource()` によるアクセス権限管理
  - _Requirements: 1.4, 1.5_

### Phase 2: Domain Layer - Core Services

- [x] 2. 画像ローディングとキャッシュ管理
- [x] 2.1 フォルダスキャン機能の実装
  - ディレクトリ列挙のストリーミング処理
  - 対応画像形式（JPEG、PNG、HEIC、WebP、GIF）のフィルタリング
  - 最初の画像が見つかった時点で即座にコールバック
  - バックグラウンドでの残りファイルスキャン
  - スキャン処理のキャンセル機能
  - _Requirements: 1.2, 1.3, 7.1, 7.2, 7.3, 7.4, 10.2_

- [x] 2.2 画像デコードとローディング機能の実装
  - ImageIO（CGImageSource）を使用した画像デコード
  - 表示サイズに合わせたダウンサンプリング
  - 優先度付きキュー（P0=表示、P1=先読み、P2=サムネイル）によるタスク管理
  - Swift Concurrency（Task.isCancelled）によるキャンセル対応
  - 進行中タスクの重複防止
  - 巨大画像（10000x10000以上）のメモリ使用量制限
  - _Requirements: 7.1, 7.2, 7.3, 7.4, 8.1, 8.2, 8.3, 10.1, 10.3, 10.5, 11.3_

- [x] 2.3 メモリキャッシュ（LRU）の実装
  - フルサイズ画像のLRU管理（最大100枚程度）
  - キャッシュヒット時の即時返却
  - メモリ警告時のキャッシュ解放
  - デコード完了分のキャッシュ保持（キャンセル時も維持）
  - _Requirements: 8.5, 10.4_

- [x] 2.4 ディスクキャッシュ（サムネイル永続化）の実装
  - 対象フォルダ内に.aiview/サブフォルダを自動作成
  - サムネイルをJPEG形式で保存
  - キャッシュキーにファイル名・更新日時・サイズを含める
  - キャッシュ書き込み失敗時はメモリキャッシュのみで継続
  - _Requirements: 9.4, 9.5, 9.6, 9.7_

- [x] 2.5 先読み（プリフェッチ）機能の実装
  - 現在位置の前3枚/後12枚を先読み
  - 進行方向に応じた優先度調整
  - 表示対象変更時の不要処理キャンセル
  - サムネイルクリックでジャンプ時の旧位置周辺キャンセル
  - _Requirements: 8.1, 8.2, 8.3, 8.4, 10.1, 10.3_

### Phase 3: Domain Layer - Metadata & State

- [x] 3. メタデータ抽出とUI状態管理
- [x] 3.1 (P) 画像メタデータ抽出機能の実装
  - ファイル名、作成日時、画像サイズ、ファイルサイズの取得
  - PNG tEXtチャンクからのプロンプト情報抽出
  - XMPメタデータへのフォールバック
  - プロンプトとネガティブプロンプトの分離抽出
  - _Requirements: 5.2, 5.3, 5.4, 5.5_

- [x] 3.2 UI状態管理ViewModel の実装
  - 現在のフォルダ、画像リスト、選択インデックスの管理
  - プライバシーモード、情報パネル表示状態の管理
  - サムネイルカルーセル表示/非表示状態の管理
  - @Observableマクロによる自動状態監視
  - MainActorでの状態更新保証
  - エラーメッセージ状態の管理
  - _Requirements: 2.1, 2.2, 2.3, 3.1, 3.2, 3.3, 3.4, 3.5, 3.6, 4.2, 4.3, 4.4, 5.1, 5.7, 6.1, 6.2, 6.3, 6.4, 11.1, 11.4_

### Phase 4: Presentation Layer - Core Views

- [x] 4. メインウィンドウと画像表示の実装
- [x] 4.1 メインウィンドウの構築
  - SwiftUIベースのメインウィンドウ
  - メニューバーに「ファイル」→「フォルダを開く」メニュー
  - メニューバーに「最近使ったフォルダ」サブメニュー
  - フォルダ選択ダイアログの表示
  - _Requirements: 1.1, 1.5_

- [x] 4.2 画像表示ビューの実装
  - ウィンドウ内で可能な限り大きく画像を表示（アスペクト比維持）
  - ローディング中の表示状態
  - 「画像がありません」状態の表示
  - 破損画像のエラープレースホルダー表示
  - エラーアラートダイアログの実装（アクセス権限エラー、削除失敗時等）
  - 「フォルダへのアクセス権限がありません」ダイアログ
  - _Requirements: 2.1, 4.3, 11.1, 11.2_

- [x] 4.3 キーボードナビゲーションの実装
  - 右カーソルキー（→）で次の画像へ移動
  - 左カーソルキー（←）で前の画像へ移動
  - 連打時もUIをブロックせず表示切り替えが追従
  - 最初/最後の画像での境界処理（ループしない）
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 3.5_

### Phase 5: Presentation Layer - Additional Features

- [x] 5. サムネイルカルーセルとパネル機能
- [x] 5.1 サムネイルカルーセルの実装
  - LazyHStackベースの仮想化スクロール（SwiftUI）
  - 現在表示中の画像を枠線でハイライト
  - サムネイルクリックで画像をメイン表示
  - 表示範囲のサムネイルから優先的に生成
  - 1000枚以上でもスクロール/描画が重くならない
  - _Requirements: 2.2, 2.3, 2.4, 2.5, 9.1, 9.2, 9.3_

- [x] 5.2 サムネイル表示トグル機能
  - 「t」キーでサムネイルカルーセルの表示/非表示切り替え
  - _Requirements: 3.6_

- [x] 5.3 (P) 画像情報パネルの実装
  - 「i」キーで情報パネルの表示/非表示トグル
  - ファイル名、作成日時、画像サイズ、ファイルサイズの表示
  - プロンプトとネガティブプロンプトの別々表示
  - プロンプト情報のクリップボードコピー機能
  - _Requirements: 5.1, 5.2, 5.5, 5.6, 5.7_

- [x] 5.4 (P) プライバシーモードの実装
  - スペースキーでメイン画像とサムネイルを非表示
  - プライバシーモード時は黒い画面を表示
  - スペースキー再押下で通常表示に復帰
  - プライバシーモード中もカーソルキー操作は有効
  - ダイアログ表示中でもスペースキーで有効化可能（グローバルキーイベント監視）
  - _Requirements: 6.1, 6.2, 6.3, 6.4, 6.5_

### Phase 6: User Actions

- [x] 6. 画像削除機能の実装
- [x] 6.1 削除操作の実装
  - 「d」キーで現在表示中の画像をゴミ箱に移動
  - 削除後は次の画像を表示（次がなければ前の画像）
  - 最後の1枚を削除した場合は「画像がありません」状態を表示
  - 削除失敗時はエラーメッセージを表示し、画像リストを維持
  - 削除後のインデックス調整
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 11.4_

### Phase 7: Integration & Polish

- [x] 7. 統合とパフォーマンス検証
- [x] 7.1 全コンポーネントの統合
  - ViewModelとすべてのサービスの接続
  - Presentation層とDomain層の結合
  - フォルダを開いてから最初の1枚表示までのフロー統合
  - カーソルキーによる画像切り替えフローの統合
  - _Requirements: 1.1, 1.2, 1.3, 7.1, 7.2, 7.3, 7.4_

- [x] 7.2 パフォーマンス検証と調整
  - 2000枚フォルダでの初回表示時間検証（目標: 500ms以内）
  - カーソルキー連打時のフレームレート検証（目標: 60fps維持）
  - サムネイルスクロール時のメモリ使用量検証（目標: 500MB以下）
  - プリフェッチ済み画像の表示時間検証（目標: 50ms以内）
  - _Requirements: 7.1, 7.2, 7.3, 7.4, 8.1, 8.2, 8.3, 8.4, 8.5, 9.1, 9.2, 9.3_

- [x] 7.3 (P) ユニットテストの実装
  - ImageLoaderのキャッシュヒット/ミス、優先度順序、キャンセル動作テスト
  - CacheManagerのLRU eviction、ディスクキャッシュ読み書きテスト
  - MetadataExtractorのPNG tEXtパース、XMPフォールバック、プロンプト分離テスト
  - FolderScannerの拡張子フィルタリング、ストリーミング列挙テスト
  - _Requirements: 1.2, 5.3, 5.4, 8.5, 9.4, 9.5, 10.1, 10.5_
